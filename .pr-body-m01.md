## Alcance técnico

### Base de datos (`supabase/migration_marketplace_proveedores.sql`)

| Tabla | Propósito |
|---|---|
| `proveedores` | perfil (slug, negocio, región, categoría, estado, plan) |
| `proveedor_servicios` | oferta bookable (precio desde, unidad, activo) |
| `proveedor_medios` | portfolio (imagen/video, orden, portada) |
| `proveedor_solicitudes` | leads novio → proveedor |
| `proveedor_favoritos` | wishlist novio ↔ proveedor/servicio |

- RLS: público ve sólo `estado='aprobado'`; owner (`user_id`) gestiona el resto.
- Helpers SQL: `user_is_proveedor_owner`, `proveedor_es_visible`.
- Triggers: `updated_at` y contador `solicitudes_mes` (para rate-limit del plan free).
- Storage bucket `proveedor-medios` con policies.
- View `v_marketplace_tarjetas` para listado público optimizado.
- `MarketplaceServicio` legacy marcado `@deprecated` (migración gradual).

### Capa TypeScript

- `src/types/database.ts`: tipos `Proveedor`, `ProveedorServicio`, `ProveedorMedio`, `ProveedorSolicitud`, `ProveedorFavorito`, `MarketplaceTarjeta`.
- `src/lib/proveedores/`: `constants`, `slug` (`asegurarSlugUnico`), `queries` (`listarTarjetasMarketplace`, `obtenerProveedorPorSlug`), `solicitudes` (`crearSolicitud` con rate-limit), `favoritos`.

### Scripts

- `scripts/seed-proveedores-dummy.mjs` — 5 proveedores aprobados en staging (idempotente).
- `scripts/smoke-m1.mjs` — 9 checks automatizados (RLS público/owner, UNIQUE favoritos NULL-safe, triggers). **9/9 passing**.
- `scripts/linear-*.mjs` — automatizaciones de seed de backlog y actualización de estados.

### Convenciones codificadas

- Idioma de interacción: **español LATAM**.
- Nombres de DB (tablas/columnas): **español snake_case**.
- Migraciones: `supabase/migration_<dominio>.sql`.
- Externalmente "favoritos" (no wishlist).
